# AFM-0013. Clap-Derived CLI Argument Parsing

Date: 2026-04-28
Last-reviewed: 2026-05-02
Tier: D
Status: Accepted

## Related

Supersedes: AFM-0002
References: AFM-0001

## Context

AFM-0002 chose manual `std::env::args()` parsing when adr-fmt had three flags. The argument surface has grown to six modes with additional parameters (`--depth`, crate name, domain filter). Manual parsing duplicates clap concerns: mutual exclusivity, help text, version display, error formatting. The reassessment trigger in AFM-0002 has been reached.

## Decision

Parse CLI arguments using clap's derive API. A single `Cli` struct
with `#[derive(Parser)]` replaces manual `resolve_args()` logic.

R1 [9]: Build all CLI argument definitions into a single `Cli`
  struct with `#[derive(Parser)]`, eliminating manual `args()`
  iteration outside that struct — exactly one struct owns all flags
R2 [9]: Declare mutually exclusive mode flags using clap's `group`
  attribute to enforce exclusivity at parse time rather than in
  runtime match logic
R3 [9]: Map each mode flag to the internal `Mode` enum via a
  conversion step after clap parsing — at least one conversion per
  flag variant
R4 [12]: Schedule a reassessment of `adr-fmt`'s `Cli` struct when
  the argument surface requires subcommands or dynamic argument
  generation beyond clap derive capabilities — file a new AFM ADR
  when that threshold is reached

## Consequences

Help output, version display, and error messages are generated by
clap and stay synchronized with argument definitions automatically.
The clap dependency adds compile time and binary size but eliminates
the maintenance burden of manual parsing for a growing argument
surface. Shell completions become available via `clap_complete` if
needed in the future.
