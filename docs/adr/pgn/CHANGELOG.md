- breaking? N — JetStream backend adapter (`pardosa::backend::jetstream`) removes the redundant `op="connect"` telemetry that wrapped every `append` and `replay`. The NATS connection is cached lazily in the substrate (`pardosa-nats` `ensure_state`), so the real connect runs once on first append; the wrapper timed and logged each append's network round-trip a second time under a false `op="connect"` label, double-counting append latency (COM-0019:R6 bounded/honest labels; PGN-0015:R1 connect-op emission is event-conditional and a cache-hit append is not a connect event). Drops the crate-private `TelemetryOp::Connect` variant and the `"connect"` value from the `op` metric-label vocabulary; no external consumer reads this dimension (crate-private enum, `info!` log fields, not public API). `TerminalCategory::Connect` (connect-failure classification) is unaffected. A genuine first-connect span (PGN-0015:R1) is deferred — a ring-pure cold-path span needs substrate→adapter surface (PGN-0010 ground), tracked as a follow-up. Mission bead: adr-fmt-8ko81.
