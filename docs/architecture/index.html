Architecture Documentation

Three independently-derived views of the same 21-crate scope, compared only after each is complete — so the comparison itself carries evidential weight instead of being a foregone conclusion.

Scope

Exactly 21 crates across three families: gh-report (gh-report, gh-report-web-client), pardosa (pardosa, pardosa-derive, pardosa-fiber-store, pardosa-file, pardosa-nats, pardosa-read, pardosa-schema, pardosa-wire, plus the two test-support crates pardosa-mint-private-tests and pardosa-test-support-harness), and cherry-pit (cherry-pit-app, cherry-pit-core, cherry-pit-gateway, cherry-pit-merger, cherry-pit-projection, cherry-pit-storage, cherry-pit-web, cherry-pit-wq, plus the tooling crate cherry-pit-sd-viz). adr-fmt, adr-srv, architect, comment-free, and non-exhaustive-check are out of scope for this doc set — they are the governance/tooling plane, not the substrate under review.

The three views

1. Design top-down

The intended architecture, synthesized strictly from prose: READMEs, DESIGN.md/OPERATIONS.md, AGENTS.md, and ADR text. No source code, no Cargo.toml, no dependency graph is read for this view — it captures what the architecture was designed to be, independent of what shipped.

Why phase separation matters

The Design view (top-down) and the C4 views (bottom-up) are produced by independent derivations from disjoint input sets: Design reads only prose (ADRs, READMEs, design docs) and is explicitly forbidden from reading source, Cargo.toml, or the dependency graph; the C4 phases read only source, manifests, and the generated dependency graph, and are explicitly forbidden from reading ADR prose or design docs while deriving structure. Each view is a self-consistent artifact on its own terms before either one is compared to the other.

This separation exists because a single author working from all sources at once tends to reconcile discrepancies silently while writing — filling in "what the code actually does" wherever the prose is vague, or reading intent into code that has drifted. That reconciliation destroys the signal a comparison is supposed to produce. By deriving Design and C4 independently and only then diffing them in the Review phase, every discrepancy the diff surfaces is real evidence of drift (or of documentation debt) — not an artifact of the authoring process papering over the gap. gh-report's own DESIGN.md already self-documents several such gaps (its W1/W2 truth-alignment corrigenda); those are exactly the class of finding this methodology is built to surface systematically, across all three families, rather than incidentally in one file.

Read order: Design → C4 (L3 then L2 then L1, or top-down L1→L2→L3 depending on preference) → Review. The Review phase is only meaningful once both independent derivations exist; reading it first will just be a diff with nothing yet to anchor it.
